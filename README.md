# harden-sles15 — SLES 15 Security Hardening Tool

A standalone, self-contained binary for applying CIS benchmarks and security hardening to SUSE Linux Enterprise Server 15 (SP0-SP4).

## Overview

`harden-sles15` is an enterprise-grade hardening tool that:

- Runs **without dependencies** on the target host
- Uses **hardware-based authorization** via device fingerprinting
- Embeds full Python runtime and Ansible playbook
- Protects IP with layered security (binary + optional encryption)
- Supports **open mode** — no license check when no fingerprints are configured

## Target Environment

| Requirement | Value |

## License / Authorization

### How It Works

The binary validates device fingerprints against a whitelist embedded at compile time. The whitelist is defined in `authorized_fingerprints.txt` — one SHA256 hash per line.

### Modes

| Whitelist Content | Behavior |
|-------------------|----------|
| **Empty** (no hashes) | **Open mode** — all devices are allowed, no license check |
| **Non-empty** | **Locked mode** — only whitelisted fingerprints can run the binary |

### Adding a Fingerprint

1. Collect the fingerprint on the target device:

   ```bash
   ./fingerprint-collector
   ```

   Output example:
   ```
   sha256:a1b2c3d4e5f6...
   ```

2. Add the hash (with or without the `sha256:` prefix) to `authorized_fingerprints.txt`:

   ```
   # authorized device fingerprints
   a1b2c3d4e5f6...
   ```

3. Rebuild the binary:

   ```bash
   docker build --output . -f Dockerfile.build .
   ```

### Whitelist File Format

- One SHA256 hash per line
- Lines starting with `#` are comments
- Empty lines are ignored
- The `sha256:` prefix is optional

Example:

```
# Production devices
a1b2c3d4e5f6...
sha256:001122334455...

# Test devices
bbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbb
```

## Security Model

### Layer 1: Device Fingerprinting

Collects hardware identifiers from:

- `/etc/machine-id` (primary)
- `/sys/class/dmi/id/*` (fallback)

Hashed with SHA256 (crypto-grade) for deterministic output.

### Layer 2: Whitelist Check

On startup, the binary validates the device fingerprint against an embedded whitelist loaded from `authorized_fingerprints.txt` at compile time via Go's `//go:embed`.

- **Locked mode**: Only whitelisted hashes are authorized. Unauthorized devices exit with a clear error message showing the device's fingerprint hash.
- **Open mode**: When the whitelist file is empty, all devices are allowed (no license check). This is useful for development or unlicensed deployments.

### Layer 3 (Optional): Playbook Encryption

Encrypt playbook assets with AES-256-GCM using a device-specific key:

```
Key = HMAC-SHA256(fingerprint, master_secret_key)
Nonce = RSA-generated (12 bytes, crypto/rand)
```

## Hardening Levels

| Level | Description |
|-------|-------------|
| 0 | No hardening (disabled) |
| 1 | Basic: filesystem config, user permissions |
| 2 | Medium: password policy, audit logging |
| 3 | Maximum: full CIS compliance |

Set via the `--level` flag:

```bash
sudo ./harden-sles15.bin --level 1
sudo ./harden-sles15.bin --level 2
sudo ./harden-sles15.bin --level 3
sudo ./harden-sles15.bin --level 0    # disabled

# Default is level 1 if --level is omitted
sudo ./harden-sles15.bin
```
