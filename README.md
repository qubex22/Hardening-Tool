# harden-snes15 — SLES 15 Security Hardening Tool

A standalone, self-contained binary for applying CIS benchmarks and security hardening to SUSE Linux Enterprise Server 15 (SP0-SP4).

## Overview

`harden-snes15` is an enterprise-grade hardening tool that:

- Runs **without dependency** on the target host
- Uses **hardware-based authorization** via device fingerprinting
- Embeds full Python runtime and Ansible playbook
- Protects IP with layered security (binary + optional encryption)

## Target Environment

| Requirement | Value |
|-------------|-------|
| OS | SLES 15 SP0-SP4 |
| Architecture | x86\_64 only |
| Privileges | Requires `sudo` |
| Internet | Not required |

## Quick Start

### Build the Binary

**Using Docker (recommended — no SLES license needed):**

```bash
docker build --output . -f Dockerfile.build .
```

This produces `hardening-snes15.bin` in the current directory.

**Using the build script (requires Go 1.21+):**

```bash
# Install Go 1.21+ first, then:
chmod +x build.sh
./build.sh
```

### Run on Target Device

```bash
sudo ./hardening-snes15.bin
```

## Fingerprint Authorization

To authorize a device:

1. Collect fingerprint (on target device):

   ```bash
   ./fingerprint-collector > customer-fingerprint.txt
   ```

2. Submit `customer-fingerprint.txt` to your organization's support team

3. Add the hash to `authorizedFingerprints` in `license/license.go`

4. Rebuild the binary with updated whitelist

### Fingerprint Format

```
sha256:1a3b4c...9z
```

## Security Model

### Layer 1: Device Fingerprinting

Collects hardware identifiers from:

- `/etc/machine-id` (primary)
- `/sys/class/dmi/id/\*` (fallback)

Hashed with SHA256 (crypto-grade) for deterministic output.

### Layer 2: Whitelist Check

On startup, the binary validates the device fingerprint against an embedded whitelist. Unauthorized devices exit with a clear error message showing the device's fingerprint hash.

### Layer 3 (Optional): Playbook Encryption

Encrypt playbook assets with AES-256-GCM using a device-specific key:

```
Key = HMAC-SHA256(fingerprint, master_secret_key)
Nonce = RSA-generated (12 bytes, crypto/rand)
```

## Hardening Levels

| Level | Description |
|-------|-------------|
| 0 | No hardening (disabled) |
| 1 | Basic: filesystem config, user permissions |
| 2 | Medium: password policy, audit logging |
| 3 | Maximum: full CIS compliance |

Set via `HARDENING_LEVEL` environment variable.
